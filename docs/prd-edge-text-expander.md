## 2. 项目背景

用户在浏览器场景中存在大量重复输入行为，例如客服回复、销售跟进、招聘邀约、运营外联、售后说明、内部沟通等。这些工作通常发生在邮箱、CRM、工单系统、社交平台、在线文档和企业后台中。

当前常见解决方案存在明显问题：

- 手动复制粘贴效率低，且容易出错。
- 输入法短语能力有限，不适合复杂模板管理。
- 常用话术散落在文档、备忘录、聊天记录里，检索和调用成本高。
- 团队难以统一维护标准话术，更新无法即时同步到实际输入场景。

因此，需要一个运行在 Edge 浏览器中的插件，让用户在网页任意输入区域中，通过快捷词快速插入常用话术和模板，减少重复劳动并提升沟通一致性。

## 3. 产品目标

- 提升浏览器场景下重复文本输入效率。
- 降低复制粘贴和来回查找模板的成本。
- 帮助个人沉淀高频话术，形成可复用模板库。
- 为后续团队共享、云同步和智能推荐能力打基础。

## 4. 产品定位

本产品是一款面向浏览器办公场景的文本补全插件，核心能力是“边输入边触发”。用户无需离开当前网页，即可通过快捷词将常用话术、模板和变量内容快速插入到任意输入框中。

## 5. 目标用户

### 5.1 核心用户

- 客服人员
- 销售人员
- 招聘和 HR 人员
- 运营和商务人员
- 重度浏览器办公的管理者和个人用户

### 5.2 用户特征

- 每天大部分工作都在浏览器中完成。
- 高频输入重复或半重复内容。
- 对话术统一性和输入效率有明确需求。
- 希望工具轻量、易上手、学习成本低。

## 6. 核心痛点

- 大量重复输入浪费时间。
- 不同场景下话术容易不一致。
- 话术管理分散，调用不方便。
- 复制粘贴会打断输入流程。
- 现有工具对网页输入框的兼容性不稳定。

## 7. 核心价值

- 用快捷词在输入时直接触发常用话术。
- 不离开当前页面即可完成模板插入。
- 支持个人积累和团队标准化沉淀。
- 同时支持“直接展开”和“候选选择”两类使用方式。

## 10. 功能需求

### 10.1 片段数据结构

每个片段至少包含以下字段：

- `id`
- `title`
- `shortcut`
- `content`
- `category`
- `tags`
- `isFavorite`
- `createdAt`
- `updatedAt`
- `lastUsedAt`
- `useCount`

### 10.2 快捷词展开

用户为话术配置快捷词，例如 `/ty`、`/refund`。当用户在支持的输入区域中输入对应快捷词，并执行设定触发动作后，插件将快捷词替换为完整话术内容。

MVP 支持的触发方式：

- `Tab`
- `Space`
- `Enter`

需求要求：

- 仅替换光标前最近一次匹配到的快捷词。
- 不破坏前后文本内容。
- 插入完成后光标位置正确。
- 失败时不得破坏原始输入内容。

### 10.3 候选面板

当用户输入触发前缀（如 `/`）或输入部分快捷词（如 `/re`）时，插件在输入区域附近展示浮动候选面板。

面板要求：

- 展示匹配结果的标题和快捷词。
- 支持上下方向键切换。
- 支持 `Enter` 确认插入。
- 支持鼠标点击插入。
- 展示内容摘要，便于快速识别。

### 10.4 片段管理

插件弹窗需要提供以下管理能力：

- 新建片段
- 编辑片段
- 删除片段
- 收藏片段
- 按标题、快捷词、标签搜索
- 按分类筛选
- 查看最近使用记录

### 10.5 变量能力

MVP 支持在片段内容中使用简单变量占位符。

示例：

```text
您好 {{name}}，感谢您的咨询，我们会在 {{date}} 前给您答复。
```

需求要求：

- 插入前识别变量。
- 弹出轻量表单收集变量值。
- 确认后渲染最终文本。
- MVP 内置支持日期、时间和固定签名。

MVP 支持的变量类型：

- 文本输入
- 当前日期
- 当前时间
- 固定签名

### 10.6 从选中文本快速保存

用户在网页中选中一段文本后，可通过右键菜单选择“保存为话术片段”。

需求要求：

- 自动将选中文本带入片段内容。
- 用户补充标题和快捷词。
- 保存过程不要求跳离当前页面。

### 10.7 设置中心

设置页支持以下能力：

- 配置触发前缀，默认 `/`
- 配置触发按键
- 开启或关闭候选面板
- 配置网站黑名单
- 配置默认签名
- 默认禁用密码框和敏感输入框

### 10.8 数据导入导出

用户可以将本地片段导出为 `JSON`，也可以重新导入。

需求要求：

- 支持导出全部片段和设置。
- 导入时进行数据格式校验。
- 快捷词重复时给出冲突提示。

## 11. 用户流程

### 11.1 直接展开流程

1. 用户在输入框中输入 `/refund`。
2. 用户按下 `Tab`。
3. 系统识别快捷词。
4. 系统将 `/refund` 替换为完整退款说明。
5. 光标移动到插入内容末尾。

### 11.2 候选选择流程

1. 用户输入 `/re`。
2. 系统展示匹配到的快捷词列表。
3. 用户通过键盘或鼠标选择目标项。
4. 用户确认插入。
5. 系统插入选中的话术内容。

### 11.3 变量填写流程

1. 用户触发包含变量的模板。
2. 系统弹出轻量变量表单。
3. 用户填写所需字段。
4. 系统生成最终文本。
5. 系统将渲染后的文本插入输入区域。

### 11.4 保存片段流程

1. 用户在网页中选中一段文字。
2. 用户通过右键菜单点击“保存为话术片段”。
3. 系统弹出快速保存表单。
4. 用户填写标题和快捷词。
5. 系统保存到本地片段库。

## 14. 非功能需求

### 14.1 兼容性

- 支持最新正式版 Edge
- 支持 Chromium 内核运行环境
- 支持 `input`、`textarea`、`contenteditable`
- 富文本编辑器需要单独做重点验证

### 14.2 性能

- 输入监听不得产生明显卡顿
- 快捷词匹配目标响应时间小于 `100ms`
- 候选面板展示目标时间小于 `150ms`

### 14.3 安全与隐私

- 默认本地存储用户片段
- 默认不上传用户输入内容
- 权限申请遵循最小必要原则
- 敏感输入区域默认禁用

### 14.4 稳定性

- 插件不得影响用户正常输入
- 插入失败时原始文本保持不变
- 遇到不支持的输入框时可平稳降级

## 15. 技术约束

- 使用 `Manifest V3`
- 使用 `background service worker`
- 使用 `content script` 负责输入监听和插入
- 使用 `popup` 提供快捷管理
- 使用 `options page` 提供设置页

核心技术难点：

- 富文本编辑器和 `contenteditable` 在不同网站实现差异较大。
- 部分前端框架使用受控输入，需要正确派发输入事件。
- 光标位置计算和候选面板定位存在兼容性问题。
- 触发键可能与网站原生快捷键冲突。

## 16. 权限建议

- `storage`
- `activeTab`
- `scripting`
- `contextMenus`
- `host_permissions`

权限设计需要在兼容性和用户信任之间平衡。开发阶段可适当放宽，正式版本需尽量收敛权限范围。

## 17. 风险与应对

### 17.1 输入兼容性风险

风险：

- 部分网站使用自定义编辑器或嵌套 iframe，控制难度高。

应对：

- 优先支持原生输入框。
- 建立兼容性测试清单。
- 增加降级策略和黑名单机制。

### 17.2 快捷键冲突风险

风险：

- 触发键可能与网站快捷键或用户输入习惯冲突。

应对：

- 提供可配置触发方式。
- 支持按站点禁用。
- 默认使用更安全的触发配置。

### 17.3 隐私顾虑风险

风险：

- 用户可能担心插件读取其输入内容。

应对：

- 默认本地存储。
- 明确解释权限用途。
- 默认排除敏感输入框。

## 18. MVP 验收标准

- 用户可以在插件中创建、编辑、删除片段。
- 用户可以在常见网页输入框中触发快捷词展开。
- 用户可以在弹窗中搜索并插入片段。
- 用户可以通过候选面板选择快捷词。
- 用户可以配置触发方式。
- 用户可以导入和导出本地数据。
- 用户可以使用至少一个带变量的模板。
- `input`、`textarea`、`contenteditable` 至少各完成一个可用示例。

## 19. 成功指标

- 激活率：安装后 24 小时内创建至少 1 个片段的用户占比
- 复用率：活跃用户日均触发片段次数
- 效率收益：节省字符数和估算节省时间
- 留存率：7 日活跃留存
- 内容深度：人均片段数
- 产品质量：插入成功率和错误率
